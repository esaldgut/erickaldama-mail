## v0.3 — Render rico (el alcance a ejecutar AHORA)

### Objetivo
El reader del correo muestra el body con **imágenes** y **HTML sanitizado**, en terminal, **sin filtrar privacidad**
(tracking pixels bloqueados) y **sin fallar** si faltan binarios externos.

### Decisiones de diseño (cerradas en brainstorm)

**D1 — Terminal primario, web secundario.** El render rico vive en la terminal (coherente con el stack nvim/tmux). El
modo web (`mail read --web`, servidor local + browser) se **difiere a v0.3.1**. v0.3 = solo terminal.

**D2 — Imágenes on-demand con `-f symbols` (no inline-siempre, no pixel-graphics en v0.3).** El reader muestra el body
en markdown (glamour) por default. Una **tecla explícita `i`** (verificado libre en el reader — no choca con J/K/r/s/Esc)
renderiza las imágenes con chafa en ese momento. **chafa se invoca en modo `-f symbols`** (half-blocks ANSI), NO pixel
graphics. Razón doble: (a) symbols es texto ANSI que **SOBREVIVE el cambio de pane de tmux** (resuelve la limitación
dura — findings 20 §A-2 / 22 §A-2); (b) no necesita `--passthrough tmux` (que en iTerm2 NO es automático — verificado:
chafa auto solo activa con Kitty). Pixel graphics (`-f iterm` + passthrough) = v0.4+.

**D3 — Bloquear imágenes remotas por default + flag consciente.** Los tracking pixels (img src=http remotos) filtran
IP/apertura — vector #1 (findings doc 20 §B-2). Por default: solo imágenes **cid:** (inline, ya en el MIME, sin red).
Las remotas → placeholder `[imagen remota bloqueada]`. Un flag (`--load-remote`) o tecla `R` (verificado libre en el
reader — solo `r` minúscula está usada para reply) las carga si el usuario confía (patrón Apple Mail "Block All Remote
Content"). **Los bytes de imagen cid: vienen de `Parsed.InlineImages` (MIME), NO del HTML sanitizado** (findings 22 §B-6).

**D4 — Sanitización SIEMPRE primero (orden obligatorio).** ⚠️ CORRECCIÓN del audit (findings 22 §B-1, verificado
EJECUTANDO bluemonday): **NO usar `UGCPolicy()` directo para `<img src>`** — UGCPolicy → AllowImages → AllowStandardURLs
PERMITE http/https (deja pasar tracking pixels) y NO incluye cid: (borra las imágenes inline). La base UGCPolicy SÍ es
correcta para todo lo demás (script/on*/iframe/style/object/embed/javascript:/data: → bloqueados, verificado). Pipeline:
```
1. Parsear (enmime/v2 — ya está; añadir InlineImages — ver Arquitectura)
2. Pre-procesar el HTML crudo (golang.org/x/net/html, YA en go.mod): recorrer <img>, si src=http/https →
   reemplazar por placeholder [imagen remota bloqueada] (salvo allowRemote=true); recolectar CIDRefs + BlockedRemotes
3. Sanitizar: bluemonday.UGCPolicy() como BASE + AllowURLSchemes("cid") y restringir img.src a solo cid (sin http/https)
4. CSS: <style>/style=/<link> ya bloqueados por UGCPolicy (verificado)
```
Firma: `func SanitizeHTML(rawHTML string, allowRemote bool) (SanitizeResult, error)` con
`SanitizeResult{ HTML string; CIDRefs []string; BlockedRemotes []string }`.
Bloquear SIEMPRE: `<script>`, `on*`, `javascript:`, `<iframe>`, `<object>`, `<embed>`, remotas (default), `<style>`/`style=`/`<link>`, `data:`.

**D5 — Degradar con gracia si faltan deps externas.** chafa es un binario externo (no viene con el cliente Go). Al
arrancar, detectar si chafa existe en PATH. Si está: render de imágenes. Si NO: mensaje claro `[chafa no instalado —
brew install chafa para ver imágenes]` y el cliente sigue funcionando con texto. **El cliente NUNCA falla por una dep
externa ausente.** Documentar en README.

**D6 — Fix del ancho de glamour (adelantado de v0.4).** Hoy `internal/message/render.go:27` usa `glamour.Render(md,
"auto")` con **ancho fijo 80** (verificado en el código de glamour). Migrar a `glamour.NewTermRenderer(
glamour.WithAutoStyle(), glamour.WithWordWrap(width))` (firmas confirmadas v0.9.1; retorna `(*TermRenderer, error)` →
manejar el error). **Fallback a 80 cuando width==0** — `WithWordWrap(0)` produce output ROTO (verificado), NUNCA pasar 0.
La firma cambia: `RenderRich(p *Parsed, width int) (string, error)` → rompe 2 call-sites (`model.go:275`,
`render_test.go:17`) — el plan los lista. El TUI guarda el **markdown CRUDO** (`rawMD`) + `termWidth` y re-renderiza en
cada resize.

### Arquitectura v0.3 (componentes — corregida por el audit)

```
internal/message/
  parse.go     (MODIF B-2) — añadir InlineImage{ContentID,ContentType,Data} + Parsed.InlineImages; iterar
                 env.Inlines Y env.OtherParts (Gmail/Outlook multipart/related), filtrar image/*; cierra el TODO v0.1
  render.go    (MODIF D6) — RenderRich(p, width int) con NewTermRenderer+WithWordWrap; fallback 80; manejar error
  sanitize.go  (NUEVO) — SanitizeHTML(rawHTML, allowRemote bool) (SanitizeResult, error); preproceso x/net/html +
                 policy UGCPolicy afinada (img.src solo cid:); devuelve HTML+CIDRefs+BlockedRemotes
internal/render/        (NUEVO paquete)
  images.go    — exec.LookPath("chafa"); RenderImage(ctx, imgData []byte, w,h int) string via
                 exec.CommandContext(ctx,"chafa","--size","WxH","-f","symbols","-") + Stdin=bytes.Reader (NO temp file);
                 timeout 5s; ExitError → placeholder (SAN-4b); chafa ausente → mensaje (SAN-4)
cmd/mail/main.go        — read: default RenderPlain (pipe-friendly); flag --rich activa SanitizeHTML+RenderRich con
                 ancho via x/term.GetSize (D6) + placeholders de remotas; flag --load-remote (solo con --rich) carga remotas
cmd/mail-tui/model.go   — AÑADIR termWidth int + rawMD string + case tea.WindowSizeMsg (B-3); tecla 'i' = imágenes
                 on-demand (symbols); tecla 'R' = toggle load-remote (re-sanitiza+re-renderiza desde el HTML crudo)
```

### Invariantes de seguridad (a testear)
- **SAN-1:** tras SanitizeHTML, el HTML NO contiene `<script>`, `on*=`, `javascript:`, `<iframe>`, `<style>`, `style=`, `<link>`.
- **SAN-2:** las imágenes remotas (src=http) NO sobreviven por default → tras SanitizeHTML(allowRemote=false), NINGÚN
  `<img>` con src http/https permanece (la remota se reemplazó por el placeholder de texto; solo sobreviven `<img>` con
  src cid:). Test: HTML con img remota → assert que la salida NO contiene "http://"/"https://" en ningún src, y SÍ contiene
  el texto placeholder. Con allowRemote=true, la img remota sí permanece (escape consciente).
- **SAN-3:** chafa se invoca con `exec.CommandContext` argv-slice (NUNCA `sh -c`). La imagen va por **stdin**
  (`cmd.Stdin = bytes.NewReader`), NO por la línea de comando ni archivo temp.
- **SAN-4:** chafa ausente (LookPath falla) → mensaje claro, NO crashea.
- **SAN-4b** (nuevo, audit A-4): chafa presente + imagen INVÁLIDA (exit 2) → `exec.ExitError` manejado → placeholder
  `[imagen no reconocida — formato inválido]`, NO propaga el error crudo.
- **SAN-5** (nuevo): los bytes de imagen cid: provienen de `Parsed.InlineImages` (MIME decodificado), NUNCA del HTML.

### Fixture de test sin NDA (audit)
`testdata/` con un `.eml` sintético: `<img src="cid:test@x">` + parte MIME `Content-ID: <test@x>` base64 de un PNG 1x1
+ una `<img src="http://remote/track.png">`. Permite testear SAN-1..SAN-5 sin tocar correos reales (NDA).

## v0.5 — Cache local (visión, NO se ejecuta en v0.3)

- Índice de **headers solamente** (from/subject/date/s3Key/mailbox — NUNCA el body → reduce superficie NDA) en

  cursor del SK más reciente. WAL para que la TUI lea sin bloquear el sync. `mail ls`/búsqueda instantáneos offline.

## Definition of Done (v0.3)
1. `go build ./... && go test -count=1 ./...` verde.
2. SAN-1..SAN-5 testeados (sanitización, no-http-en-img-remotas, argv-slice/stdin, degradación, imagen-inválida, cid:-desde-MIME) usando el fixture `.eml` sintético.
3. El TUI muestra el body con ancho correcto (fix D6) + placeholders de remotas; `mail read --rich` (CLI) hace lo mismo
   con ancho via x/term.GetSize. `mail read` sin --rich sigue en RenderPlain (pipe-friendly).
4. La tecla `i` (TUI) renderiza imágenes cid: con chafa `-f symbols` (o degrada con mensaje claro si falta chafa).
5. `--load-remote` (CLI, con --rich) / tecla `R` (TUI) cargan las remotas con escape consciente.
6. Gate NDA sobre el diff. Smoke: render de un correo real con imagen (gate humano para correos reales — NDA).
7. README documenta: deps opcionales (chafa), la política de privacidad (remotas bloqueadas), el flujo de imágenes.
8. PR a develop, CI verde.
