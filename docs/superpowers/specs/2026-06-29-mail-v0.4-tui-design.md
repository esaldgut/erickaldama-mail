## Objetivo

Una sola frase: convertir el TUI de 3 pantallas-excluyentes a un **layout de 2 paneles vivos** (lista ⟷ reader)
con focus alternable, lista con filtro fuzzy nativo, reader con scroll real (viewport), y feedback async (spinner) —
cerrando el papercut de navegación H1 del smoke E2E.

### Firmas verificadas compilando — el riesgo v1-vs-v2 RESUELTO

El riesgo crítico era que los ejemplos públicos del pager usan la API de bubbles **v2 beta**; nosotros usamos
**v1.0.0**, cuya API difiere. El compilador rechazó la API v2 (evidencia binaria en finding 24). Las firmas v1.0.0:

```go
// viewport v1.0.0 — CONSTRUCTOR CON ARGS (v2's viewport.New() falla: "not enough arguments")
vp := viewport.New(width, height int)        // (Model)
// DIMENSIONAR POR CAMPOS PÚBLICOS — NO setters (v2's SetWidth falla: "has no field or method SetWidth")
vp.Width = w;  vp.Height = h
vp.SetContent(s string)
vp.LineDown(n); vp.LineUp(n); vp.HalfViewDown(); vp.HalfViewUp(); vp.GotoTop(); vp.GotoBottom()
p := vp.ScrollPercent() // float64 0..1 ; vp.AtTop() bool ; vp.AtBottom() bool
vp.MouseWheelEnabled = true // + tea.WithMouseCellMotion() en NewProgram
m.viewport, cmd = m.viewport.Update(msg) // Update retorna (Model, tea.Cmd) — reasignar
// keymap propio en Update: j/k/space/f/d/u/h/l/b/pgup/pgdn — el viewport navega solo cuando tiene foco

// spinner v1.0.0 — Tick es MÉTODO (el spinner.Tick de paquete está Deprecated → routing roto)
sp := spinner.New(spinner.WithSpinner(spinner.Dot))  // (Model)
// Init/Batch reciben sp.Tick SIN paréntesis (es func() tea.Msg, asignable a tea.Cmd)
m.spinner, cmd = m.spinner.Update(msg) // se auto-rearma ante spinner.TickMsg
sp.View() string

// list v1.0.0 — usamos NewDefaultDelegate → items implementan list.DefaultItem (3 métodos)
type DefaultItem interface { Item; Title() string; Description() string } // Item = FilterValue() string
l := list.New(items []list.Item, delegate list.ItemDelegate, width, height int) // (Model)
l.SetSize(w, h int)
l.SelectedItem() list.Item  // opera sobre VisibleItems()+Index() → devuelve el FILTRADO correcto
l.SettingFilter() bool      // == FilterState()==list.Filtering ; true → no robar teclas al layout
l.SetShowTitle(false); l.SetShowStatusBar(false) // embeber limpio en panel estrecho
m.list, cmd = m.list.Update(msg) // (Model, tea.Cmd); navega j/k + filtro / + paginación internamente
```

## Decisiones de diseño (cerradas en el brainstorm)

### D1 — Layout: 2 paneles simultáneos + focus alternable con Tab (estilo mutt/aerc)
Lista (izq ~35%) y reader (der ~65%) visibles a la vez vía `lipgloss.JoinHorizontal(lipgloss.Top, listPane, readerPane)`.
Un campo `focus` (`focusList`/`focusReader`) enruta las teclas. **Tab** alterna. El panel con foco lleva border
resaltado (`RoundedBorder` + `BorderForeground(AdaptiveColor{Light,Dark})`); el otro, border tenue. Cierra H1 de raíz:
ves lista + contenido juntos. (Descartadas: pantallas-alternas — no cierra H1; reader-preview — añade un modo extra.)

### D2 — Los 4 modos del modelo (reemplazan los 3 viewState actuales)
| Modo | Pantalla | Teclas |
|---|---|---|
| `modeBrowse` | 2 paneles (list + reader) | Tab alterna focus; teclas → panel con foco |
| `modeFilter` | 2 paneles, list filtrando | **TODAS** las teclas → `list` (regla D3) |
| `modeComposer` | overlay full-screen | `handleComposeKey` (de v0.2, intacto) |
| `modeConfirm` | overlay de confirmación | y/n |

### D3 — Regla de oro del focus (cierra el NO VERIFICADO #1 de Fase 0)
Antes de enrutar cualquier `KeyMsg`: si `m.list.SettingFilter()` es true → la tecla va **entera** a `m.list.Update()`
(el usuario está tipeando el filtro). Evita que Tab/Enter/letras se las robe el layout durante el filtrado. Los 3
estados: `Unfiltered` (navega normal), `Filtering` (robar teclas), `FilterApplied` (filtro fijo, el padre vuelve a
navegar entre paneles).

### D4 — Composer = overlay pantalla-completa (4º modo, separado del layout de lectura)
Al presionar `c` (componer) o `r` (responder), el composer toma toda la pantalla como overlay; al enviar o cancelar
(Esc) vuelve a los 2 paneles. El `Tab` interno del composer (navega campos To/Cc/Bcc/Subject) **NO choca** con el Tab
de focus entre paneles porque solo un modo está activo a la vez. El composer de v0.2 queda casi intacto, solo se
extrae a su propio archivo.

### D5 — Lista: Title=Subject, Desc=From+fecha, FilterValue=From+Subject
Cada `mailbox.Header` se vuelve un `messageItem` que implementa `list.DefaultItem`:
- `Title()` → `Subject` (lo que identifica el correo al escanear)
- `Description()` → `From + " · " + fechaCorta` (contexto sin ruido)
- `FilterValue()` → `From + " " + Subject` (el filtro `/` busca por quién o por tema)

Usa el `DefaultDelegate` verificado en Fase 0 (cero riesgo nuevo). El delegate custom de 1-línea (densidad) se difiere.

### D6 — Spinner en las 3 operaciones async
Un bool `loading` + `spinner.Tick` por operación. Las 3 candidatas, cada una pone `loading=true` y devuelve una
`tea.Cmd`; el mensaje de retorno apaga el spinner:

| Disparo | Cmd async | Mensaje retorno |
|---|---|---|
| Selección en lista cambia | carga body S3 + parse | `bodyLoadedMsg{parsed}` |
| Tecla `i` (reader) | render chafa imágenes | `imagesRenderedMsg{blobs}` |
| Ctrl-S (composer) | envío SES | `sentMsg{messageID}` |

### D7 — Mouse + WithMouseCellMotion
`tea.NewProgram(m, tea.WithAltScreen(), tea.WithMouseCellMotion())` — se **añade** `WithMouseCellMotion()` (hoy falta;
el viewport lo necesita para scroll de rueda) + `vp.MouseWheelEnabled = true`. Solo scroll de rueda; no drag/click en items (YAGNI).

### D8 — Cierra la deuda v0.3 (WB-4 + WB-2)
- **WB-4** (migrar scrollOffset+images al viewport): se cierra solo — el viewport REEMPLAZA scrollOffset, y el render
  de imágenes (tecla `i`) se integra como contenido del viewport (re-`SetContent` con las imágenes).
- **WB-2** (tests del TUI para R/imagesRenderedMsg): se cierra con los tests del nuevo flujo (ver Testing).

## Flujo de datos — el ciclo Update

```
Update(KeyMsg):
  1. modeComposer/modeConfirm?  → handleComposeKey / y-n        (overlay manda)
  2. m.list.SettingFilter()?    → TODO → m.list.Update()        (D3: no robar teclas)
  3. teclas globales (Tab,q,c,r,s,a) → switch ; Tab → alterna m.focus
  4. según m.focus:
       focusList   → m.list.Update(msg) ; si cambió selección → loadBody (async, loading=true)
       focusReader → m.viewport.Update(msg)                     (j/k/pgup internos)
```

**Reader load (`bodyLoadedMsg`):** sanitizar (P-5, intacto v0.3) → `RenderRich(clean, readerWidth)` →
`viewport.SetContent(body)` → `viewport.GotoTop()` → `loading=false`. `readerWidth` = ancho del panel derecho
**menos** `Style.GetFrameSize()` (el border), NO `termWidth` completo.

**Resize (`WindowSizeMsg`):** recalcular anchos de ambos paneles → `list.SetSize(lw, h)` →
`viewport.Width/Height = ...` → re-`RenderRich` al nuevo `readerWidth`. Guard `vpReady`: no renderizar paneles
hasta el primer resize (NO VERIFICADO #3 de Fase 0).

## Manejo de errores (continuidad con v0.3)
- Falla carga S3 → `errMsg` → status line en footer, `loading=false`, reader queda con contenido previo o vacío (no crashea).
- Falla `SanitizeHTML` → fallback a texto plano (P-5/WB-3 de v0.3, intacto).
- chafa ausente / imagen inválida → placeholder (SAN-4/SAN-4b de v0.3, intacto).
- `WindowSizeMsg` inicial no llega → guard `vpReady` evita render con dimensiones 0.
