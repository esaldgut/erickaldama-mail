## 5. Seguridad y privacidad

- **BCC nunca en el header MIME** â€” solo en `SendRawEmailInput.Destinations`. Es el invariante de privacidad central;
  un test lo asegura. Si el BCC se filtrara al header, todos los destinatarios verĂ­an las copias ocultas.
- **Menor privilegio intacto**: el ls multi-mailbox usa Query (no Scan) â†’ NO requiere ampliar la policy de
  `mail-client-read`. Cero cambio de IAM.
- **config.toml**: no contiene secretos (solo direcciones + nombres de profile). Los profiles AWS siguen en `~/.aws`.
- **Repo pĂşblico**: gate NDA sobre todo el output. El config.toml de ejemplo en docs usa direcciones publicables
  (`erick@erickaldama.com` es publicable).
