# SP-2 — Identidad SES + envío: Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `erickaldama.com` send authenticated email (DKIM/SPF/DMARC pass) via SES v2, provisioned entirely in CDK-Go on the SP-1 foundation — identity + DKIM + custom MAIL FROM + DMARC + configuration set (EventBridge) + reputation alarms + a scoped send IAM role — verified live with a Mailbox-Simulator smoke, all inside the sandbox.

**Architecture:** A second CDK-Go stack `SendingStack` (account `367707589526`, region `us-east-1`) built TDD against the synthesized template. The SP-1 hosted zone (`Z023932911KA6S98A6ZRW`) is imported by attributes (no AWS call). `awsses.EmailIdentity` with `Identity_PublicHostedZone` auto-publishes the 5 records (3 DKIM CNAME + MAIL FROM MX + MAIL FROM SPF TXT); only the DMARC TXT is added manually. The config set sends BOUNCE/COMPLAINT to EventBridge (default bus) → a Rule → SNS topic; feedback-forwarding is OFF to avoid duplicate notifications. Two reputation alarms (treatMissingData IGNORE). A `mail-send` ManagedPolicy (scoped to the identity ARN + `ses:FromAddress` condition) on an assumable `mail-sender-role`. The exec-policy gains `events:*` (anticipated, not discovered-in-deploy as SP-1's ssm gap). The agent writes/synths/tests read-only; the human deploys + smokes out-of-band with SSO Admin; the agent verifies (including an async DKIM polling gate before the smoke).

**Tech Stack:** Go 1.26.4 + `aws-cdk-go/awscdk/v2 v2.258.1` (packages `awsses`, `awsroute53`, `awsevents`, `awseventstargets`, `awssns`, `awscloudwatch`, `awsiam`) + CDK `assertions` + `go test` (offline template asserts); AWS IAM JSON (exec-policy + mail-send mirror); `aws sesv2 get-email-identity` polling (DKIM gate); `cdk deploy` (human, SSO Admin); SES Mailbox Simulator (smoke).

**Source spec:** `docs/superpowers/specs/2026-06-11-sp2-ses-send-design.md` (commit c3da31f). Audit findings: `~/.claude/plans/email-project-research/11-sp2-audit-findings.md`.

**Account/region:** ErickSA `367707589526`, `us-east-1`. Apply `aws-cli-pre-flight-canonical` before any `aws` command.

**Live ground truth (verified 2026-06-10):** SES us-east-1 SANDBOX (ProductionAccessEnabled=false), 200/day @1/s, 0 sent, suppression BOUNCE+COMPLAINT already ON, zero identities/config-sets. Hosted zone `Z023932911KA6S98A6ZRW` (erickaldama.com) live.

**All CDK-Go code below was COMPILED against awscdk v2.258.1** (audit agents built + synthesized + asserted the whole stack). Use it verbatim. Key signature facts the audit corrected: `Identity` (not `Domain`) field; `MailFromBehaviorOnMxFailure` (not `BehaviorOnMxFailure`); `FeedbackForwarding` is an **EmailIdentityProps** field (not config-set); EventBridge factory is `EventDestination_EventBus` (not `_EventBridge`); targets import is `awseventstargets` (one token); an extra `AWS::SNS::TopicPolicy` is auto-emitted by the SNS target.

func NewSendingStack(scope constructs.Construct, id string, props *awscdk.StackProps) awscdk.Stack {

	return map[string]*string{
		"Project":    strptr("erickaldama-mail"),
		"Subproject": strptr("SP-2"),
		"ManagedBy":  strptr("CDK-Go"),
	}
}

### Task 9: DKIM async-wait gate script

- [ ] **Step 1: Create the script** `iam/ses-dkim-wait.sh`

```bash
#!/usr/bin/env bash
# ses-dkim-wait.sh — the async DKIM verification gate (SP-2).
# A `cdk deploy` of SendingStack returns CREATE_COMPLETE with DKIM still PENDING — verification is
# asynchronous (minutes with Route53 same-account, up to 72h worst case). This polls until DKIM
# is SUCCESS and the identity is verified for sending, BEFORE the smoke. Read-only. Safe to re-run.
set -euo pipefail

IDENTITY="erickaldama.com"
PROFILE="${SES_PROFILE:-mail-readonly}"   # agent reads with mail-readonly (ses:Get* is allowed)
MAX_ATTEMPTS="${MAX_ATTEMPTS:-30}"        # 30 × 60s = 30 min cap
SLEEP_SECS="${SLEEP_SECS:-60}"

echo "=== waiting for DKIM=SUCCESS on $IDENTITY (profile $PROFILE) ==="
attempt=0
while [ "$attempt" -lt "$MAX_ATTEMPTS" ]; do
  attempt=$((attempt + 1))
  STATUS=$(aws sesv2 get-email-identity --email-identity "$IDENTITY" \
    --profile "$PROFILE" --region us-east-1 \
    --query 'DkimAttributes.Status' --output text 2>/dev/null || echo "ERROR")
  VERIFIED=$(aws sesv2 get-email-identity --email-identity "$IDENTITY" \
    --profile "$PROFILE" --region us-east-1 \
    --query 'VerifiedForSendingStatus' --output text 2>/dev/null || echo "ERROR")
  echo "attempt $attempt/$MAX_ATTEMPTS: DKIM=$STATUS VerifiedForSending=$VERIFIED"
  if [ "$STATUS" = "SUCCESS" ] && [ "$VERIFIED" = "True" ]; then
    echo "PASS: DKIM verified, identity ready for sending"
    exit 0
  fi
  if [ "$STATUS" = "FAILED" ]; then
    echo "FAIL: DKIM status FAILED — check the CNAMEs in the hosted zone"; exit 1
  fi
  sleep "$SLEEP_SECS"
done
echo "TIMEOUT: DKIM still not SUCCESS after $MAX_ATTEMPTS attempts (propagation can take up to 72h)"
exit 2
```
